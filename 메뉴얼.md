## 1. 프로그램 개요

테니스 클럽의 당일 참여자 관리, 복식 대진표 자동 생성, 경기 결과 등록, 점수/승률 통계를 통합 관리하는 웹 애플리케이션입니다.

### 기본 상수 및 기준값

| 항목 | 값 | 설명 |
|---|---|---|
| 기본 시작시간 | `07:00` | 경기 시작 기본값 |
| 게임 1회 소요시간 | `40분` | 한 라운드 진행 시간 |
| 총 게임 수 | `5게임` | 당일 최대 진행 게임 수 |
| 그룹 기반 게임 | `1~3게임` | 점수 그룹(A/B) 기반 대진 |
| 랭킹 기반 게임 | `4~5게임` | 전체 순위 기반 교차 대진 |
| 최소 출전 인원 | `4명` | 4명 미만이면 매치 미생성 |
| 체크인 선택지 | 시작시간 / +30분 / +60분 | 3가지 선택 가능 |
| 체크아웃 선택지 | 체크인 시각부터 30분 단위 | 최대 체크인+240분(4시간) 이내 |

## 2. 시작 및 초기화

### 2-1. 프로그램 접속

- 브라우저에서 해당 URL에 접속합니다.
- 프로그램은 자동으로 **서버(DB)** 또는 **로컬 저장소** 중 사용 가능한 곳에서 데이터를 불러옵니다.
  - `http://` 또는 `https://` 환경 → **서버(DB) 우선** 사용
  - 로컬 파일(`file://`) 환경 → **브라우저 localStorage** 사용

### 2-2. 최초 실행 — user.txt 등록

최초 실행 시 회원 목록이 비어 있으면 `user.txt` 파일을 등록해야 합니다.

**user.txt 형식:**
```
이름<TAB>베이스점수
```
예시:
```
홍길동	1200
김철수	980
이영희	1050
```
- 항목 구분은 **탭(Tab)** 문자 사용
- 그레이드 항목은 현재 버전에서 사용하지 않으며, 이름과 점수만 필요합니다

### 2-3. 로딩 화면 버튼 안내

| 버튼 | 기능 |
|---|---|
| user.txt 파일 선택 | 회원 목록을 파일에서 불러와 저장 |
| user.txt에서 사용자 정보 업데이트 | 기존 회원 정보를 파일 기준으로 갱신 |
| 서버(DB)에서 다시 불러오기 | 서버 최신 데이터로 강제 동기화 |
| 로컬 데이터 서버 업로드 | 로컬 저장 데이터를 서버에 업로드 |
| 상태 파일로 저장 (state.json 다운로드) | 현재 전체 데이터를 JSON 파일로 백업 |

## 3. 달력 화면

### 3-1. 달력 탐색

- 상단 **◀ / ▶** 버튼으로 월을 이동합니다.
- 경기가 예정된 날짜(참여자가 등록된 날짜)는 달력에 표시됩니다.
- 날짜를 클릭하면 **당일 참여자 화면**으로 이동합니다.

### 3-2. 경기 시작시간 설정 (월 단위)

- 달력 상단의 **경기 시작시간** 선택박스에서 해당 월의 기본 시작시간을 설정합니다.
- 선택 범위: `05:00` ~ `12:00` (1시간 단위)

## 4. 당일 참여자 관리

### 4-1. 참여자 목록

- 당일 화면에는 전체 회원이 목록으로 표시됩니다.
- 각 회원 항목에서 다음 3가지를 설정합니다.

| 설정 항목 | 기준 |
|---|---|
| **참여 여부** | 체크박스로 당일 출석 여부를 표시 |
| **체크인 시각** | 코트 도착 시각 (시작시간 / +30분 / +60분 중 선택) |
| **체크아웃 시각** | 퇴장 예정 시각 (체크인 이후 30분 단위, 최대 +4시간) |

### 4-2. 당일 시작시간 설정

- 화면 상단의 **당일 시작시간**에서 해당일 경기 시작시간을 개별 설정할 수 있습니다.

### 4-3. 게임 참여 가능 조건 (기준)

각 게임(1~5)에 특정 참여자가 포함되려면 다음 조건을 모두 충족해야 합니다.

```
체크인 시각 ≤ 게임 시작시각
체크아웃 시각 ≥ 게임 종료시각(게임 시작 + 40분)
```

- 게임 1 시작: 당일 시작시간 (예: 07:00)
- 게임 2 시작: 당일 시작시간 + 40분 (예: 07:40)
- 게임 3 시작: 당일 시작시간 + 80분 (예: 08:20)
- 게임 4 시작: 당일 시작시간 + 120분 (예: 09:00)
- 게임 5 시작: 당일 시작시간 + 160분 (예: 09:40)

> **예시:** 시작시간 07:00, 체크인 07:30, 체크아웃 09:00인 참여자는 게임 2(07:40~08:20)만 참여 가능합니다.

## 5. 대진표 작성 및 경기 진행

### 5-1. 대진표 작성

1. 당일 참여자 화면 하단의 **대진표 작성** 버튼을 누릅니다.
2. 관리자 암호를 입력합니다 (기본값: `1234`).
3. 암호 확인 후 대진표가 자동 생성되어 화면에 표시됩니다.

> **주의:** 대진표는 한 번 생성하면 저장되며, 재생성하려면 관리자 암호가 다시 필요합니다.

### 5-2. 대진표 화면 구성

- 각 게임(1~5)은 **게임 카드**로 표시되며, 카드에는 다음 정보가 포함됩니다.
  - 게임 번호 및 시작 시각
  - 각 매치별 팀 구성 (팀1 vs 팀2)
  - 대기자 목록
  - 팀 점수 합계

### 5-3. 경기 결과 입력

1. 각 매치에서 **승리 팀을 클릭**합니다 (클릭한 팀이 하이라이트됩니다).
2. 모든 경기의 승자를 선택한 뒤 하단 **결과 등록** 버튼을 누릅니다.
3. 확인 창에서 승인하면 점수에 즉시 반영됩니다.

> - 결과 등록은 선택된 경기만 반영됩니다 (일부 경기만 등록 가능).
> - 등록 후에는 순위/승률 화면으로 자동 이동합니다.
> - 이미 등록된 결과는 재등록 시 자동으로 이전 결과가 취소되고 새 결과로 교체됩니다.

## 6. 게임 구성 방식 — 상세 기준

### 공통 기준

- 모든 매치는 **2vs2 복식** 형태입니다.
- 같은 날 이미 같은 팀이었던 두 사람은 가능한 한 다시 같은 팀이 되지 않도록 배치합니다.
- 4명 미만인 경우 해당 게임의 매치는 생성되지 않습니다.
- 인원이 4의 배수가 아닌 경우, 초과 인원(`N % 4`명)은 **대기자**로 분류됩니다.

---

### 게임 1~3 (그룹 기반 대진)

**목적:** 비슷한 실력의 선수끼리 매치되도록 A/B 그룹을 나누어 구성합니다.

#### 기준

| 기준 항목 | 내용 |
|---|---|
| 그룹 분류 기준 | 해당 게임 참여 가능 인원을 **총점수 내림차순** 정렬 후, 상위 절반 → **A그룹(상위)**, 하위 절반 → **B그룹(하위)** |
| 대기자 선발 | 전체 인원에서 **랜덤**으로 `N % 4`명 차출 |
| 매치 구성 우선순위 | ① A그룹끼리 4명 매치 → ② A그룹 잔여 + B그룹 보충 4명 매치 → ③ B그룹끼리 4명 매치 |
| 팀 분배 방식 | 4명 안에서 같은 날 같은 팀 경험이 없는 조합 우선 선택 (최대 120회 시도) |
| 재시도 횟수 | 최대 120회 반복하여 **동일 날 팀 재구성 최소화**된 결과를 선택 |

#### 예시 (12명, 그룹별 매치 구성)

```
총점수 순위: 1위~12위
A그룹(상위): 1~6위
B그룹(하위): 7~12위

매치 1: A그룹 4명 → [A팀] vs [B팀]
매치 2: A그룹 2명 + B그룹 2명 → [A팀] vs [B팀]  ← A잔여 + B보충
매치 3: B그룹 4명 → [A팀] vs [B팀]
```

---

### 게임 4 (전체 순위 기반 교차 대진)

**목적:** 전체 인원을 순위로 나열하고, 최상위와 최하위가 같은 팀이 되어 팀 밸런스를 맞춥니다.

#### 기준

| 기준 항목 | 내용 |
|---|---|
| 정렬 기준 | 해당 게임 참여 가능 인원을 **총점수 내림차순** 정렬 |
| 대기자 선발 | 정렬 후 **하위 `N % 4`명** 자동 대기 |
| 팀 구성 규칙 | 상위 끝과 하위 끝을 쌍으로 묶어 팀 구성 |
| 매치 구성 규칙 | `[1위, 꼴찌]` vs `[2위, 2꼴찌]`, `[3위, 3꼴찌]` vs `[4위, 4꼴찌]` ... (순서대로) |

#### 예시 (10명 → 대기 2명, 출전 8명)

```
대기: 9위, 10위

매치 1: [1위 + 8위] vs [2위 + 7위]
매치 2: [3위 + 6위] vs [4위 + 5위]
```

---

### 게임 5 (Shifted 교차 대진 + 4게임 대기자 복귀)

**목적:** 4게임 대기자를 반드시 출전시키고, 4게임과 다른 팀 조합을 만들어 모든 참여자가 공평하게 플레이할 수 있도록 합니다.

#### 기준

| 기준 항목 | 내용 |
|---|---|
| 4게임 대기자 처리 | 4게임 대기자는 **반드시 5게임 출전** |
| 5게임 대기자 선발 | 4게임 출전자 중 **하위 `N % 4`명** 대기 (이들은 이미 4게임에서 뛰었으므로 공평) |
| 출전 인원 구성 | (4게임 출전자 − 5게임 대기자) + 4게임 대기자 |
| 팀 구성 규칙 | **Shifted(한 칸 밀림)** 페어링: `[1위 + 2꼴찌]` vs `[2위 + 3꼴찌]`, ... |
| 마지막 매치 처리 | 루프 후 남은 3명 + **절대 꼴찌(최하위 1명)** 를 마지막 매치에 배정 |
| 전원 출전 보장 | 위 방식으로 대기자 없이 전원 출전 가능 |

#### 예시 (10명 → 대기 2명, 출전 8명)

```
4게임 대기자: 9위, 10위
5게임 대기자(4게임 출전자 중 하위 2명): 7위, 8위
5게임 출전(8명): 1위, 2위, 3위, 4위, 5위, 6위, 9위, 10위

매치 1: [1위 + 9위(2꼴찌)] vs [2위 + 6위(3꼴찌)]
매치 2: [3위 + 5위(4꼴찌)] vs [4위 + 10위(절대 꼴찌)]
```

#### 4게임 vs 5게임 팀 구성 비교 (출전 8명 기준)

| 게임 | 매치 1 | 매치 2 |
|---|---|---|
| 4게임 | [1위+8위] vs [2위+7위] | [3위+6위] vs [4위+5위] |
| 5게임 | [1위+7위] vs [2위+6위] | [3위+5위] vs [4위+8위] |

> 5게임에서는 순위가 한 칸씩 밀려 4게임과 다른 조합이 만들어집니다.

## 7. 점수 시스템

### 7-1. 점수 구조

```
총점수 = 베이스 점수 + 경기 점수
```

| 점수 종류 | 설명 |
|---|---|
| **베이스 점수** | 회원 등록 시 부여되는 기초 점수 (user.txt 또는 회원관리에서 설정) |
| **경기 점수** | 경기 결과에 따라 누적되는 점수 (승리 시 +, 패배 시 -) |
| **총점수** | 대진표 생성 및 순위 산출에 사용되는 기준 점수 |

### 7-2. 경기 결과 점수 변동 기준

| 상황 | 점수 변동량(delta) | 기준 |
|---|---|---|
| **예상 승리** (승리팀 점수합 ≥ 패배팀 점수합) | **±1점** | 강팀이 약팀을 이기는 경우 |
| **역전 승리** (승리팀 점수합 < 패배팀 점수합) | **차이의 30%, 최소 1점** (소수점 버림) | 약팀이 강팀을 이기는 경우 |

- 승리 팀 전원 → `+delta`점
- 패배 팀 전원 → `-delta`점

**역전 승리 예시:**

```
A팀 총점수 합: 2000점, B팀 총점수 합: 2600점
B팀이 승리 (역전 승리)
차이 = 600점
delta = floor(600 × 0.3) = 180점
→ B팀 전원 +180점, A팀 전원 -180점
```

### 7-3. 결과 재등록 시 처리

- 이미 등록된 결과가 있는 날에 **재등록**하면, 이전 결과가 자동으로 취소(점수 원복)된 후 새 결과가 적용됩니다.

## 8. 순위 및 승률 화면

### 8-1. 전체 순위 탭

- 전체 회원을 **총점수 내림차순**으로 표시합니다.
- 각 회원의 총 경기 수, 승수, 패수, 승률을 확인할 수 있습니다.
- 회원을 클릭하면 **해당 회원 기준 상대별 전적** 상세 화면으로 이동합니다.

### 8-2. 상대별 승률 탭

- 기준 사용자를 선택하면, 해당 사용자가 각 상대와의 대전 수·승패·승률을 확인할 수 있습니다.
- 대전 기록이 없는 상대는 목록에서 제외됩니다.

## 9. 회원관리

### 9-1. 접근

- 하단 네비게이션 **회원관리** 탭을 누릅니다.
- 관리자 암호 입력 창이 표시됩니다.
- **기본 암호: `1234`**

### 9-2. 회원 추가

1. **회원 추가** 버튼 클릭
2. 이름, 베이스 점수 입력
3. **저장** 버튼 클릭

### 9-3. 회원 수정

1. **회원 수정** 버튼 클릭
2. 수정할 회원 선택 후 정보 변경
3. **저장** 버튼 클릭

### 9-4. 회원 삭제

## 1. 기획 배경 및 목적

테니스 동호회는 매주 같은 멤버들이 반복 경기를 진행한다. 수기로 대진을 짤 경우 다음과 같은 문제가 발생한다.

- 관리자가 인원을 세고 직접 조합을 작성하는 데 시간이 걸린다
- 실력 편차를 고려하지 않으면 같은 팀이 반복되거나 경기 불균형이 생긴다
- 참여 여부, 도착 시각, 조기 퇴장이 다양하여 수동 관리가 어렵다
- 경기 결과와 승률을 별도로 기록하지 않으면 실력 변화를 추적할 수 없다

이 시스템은 위 문제를 해결하기 위해 설계된 **웹 기반 테니스 클럽 통합 관리 도구**이다.

## 3. 게임 구성 철학 및 알고리즘 설계 의도

### 3-1. 왜 5게임인가

- 테니스 동호회 평균 활동 시간이 3~4시간이며, 게임당 40분 기준 5게임이 가장 적합하다.
- 1~3게임(그룹 기반) + 4~5게임(전체 교차) 구조로 **전반부는 레벨 매칭, 후반부는 전원 통합** 방식을 채택했다.

### 3-2. 1~3게임: 그룹 기반 대진

- 초반에는 실력이 비슷한 사람끼리 경기해 **각자 실력에 맞는 난이도**를 경험하도록 한다.
- 총점수 기준 상위 절반(A그룹)과 하위 절반(B그룹)으로 분리 후 같은 그룹 내에서 매치를 구성한다.
- 인원 수가 맞지 않을 경우(A 잔여 발생) B그룹에서 보충해 혼합 매치도 허용한다.
- **같은 날 같은 팀 방지** 로직을 120회 시도해 가능한 한 다양한 팀 조합을 보장한다.

### 3-3. 4게임: 전체 교차 대진 (Best+Worst)

- 후반부 첫 게임으로, 전체 인원을 하나의 리그로 통합해 레벨 경계를 허문다.
- 팀 구성 원칙: **1위 + 꼴찌 = 한 팀, 2위 + 2꼴찌 = 한 팀** (Best+Worst 방식).
- 이 방식은 팀 간 점수 합계를 균등하게 만들어 **경기 긴장감과 균형**을 동시에 달성한다.
- 대기자는 하위 `N%4`명으로 자동 선정하여, 5게임에서 반드시 출전하게 된다.

### 3-4. 5게임: Shifted 교차 대진 + 전원 출전 보장

- 4게임 대기자가 5게임에서 반드시 출전하도록 설계했다.
- 5게임 대기자는 4게임 출전자 중 하위 `N%4`명이 맡아 **전체적으로 대기 부담이 고르게 분산**된다.
- 팀 구성 원칙: **1위 + 2꼴찌 = 한 팀, 2위 + 3꼴찌 = 한 팀** (Shifted 방식).
  - 4게임에서 꼴찌와 팀이었던 1위가 5게임에서는 2꼴찌와 팀이 되어 **매번 다른 파트너**를 경험한다.
- 마지막 매치에 절대 꼴찌를 추가로 편입하여 **남는 사람이 없도록 보장**한다.
- 결과적으로 5게임에 걸쳐 모든 참여자가 **최소 4게임, 최대 5게임** 출전하는 균형이 달성된다.

## 4. 점수 시스템 설계 의도

### 4-1. 베이스 점수

- 회원 등록 시 부여되는 초기 실력 지수이다.
- user.txt 등록 또는 관리자 수동 설정으로 초기 레벨을 반영한다.
- 대진표 생성 시 그룹 배정 및 순위 산출의 기준이 된다.

### 4-2. 경기 점수 — 업셋 보상 구조

- 단순 승패에 1점만 부여하면 실력 차이가 점수에 반영되지 않는다.
- 이를 해결하기 위해 **역전 승리 시 추가 보상** 구조를 도입했다.
  - **예상 승리:** ±1점 (강팀이 이겨도 소폭 변동)
  - **역전 승리:** 점수 차이의 30%, 최소 1점 (약팀이 이기면 큰 보상)
- 이 구조는 **약자에게 도전 동기**를 부여하고, 시간이 지남에 따라 **자연스럽게 실력 순으로 점수가 수렴**하도록 유도한다.
